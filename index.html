    Tomato is here!

    Tomato is the ORM framework for the modern development world. When using Tomato, you will never generate nor write entity classes. You will not write repository classes. You will not write service classes. You will not write REST services. You will not need to configure a persistence context. You will not need to work with EntityManager instances.
    Tomato gives you more performance and more flexibility in less lines of code and classes. How many less classes - 98% less classes, 98% less code. Regardless of the number of tables you are accessing, using Tomato, you will use one Entity class and one REST service. Period. These 2 classes provide more bang per line of code than Spring Data or other comparable solutions. Right out of the box, you can read and write single rows, row sets and data hierarchies to and from tables in any schema knowing nothing more than the schema and table name. Tomato is also completely thread safe without any modifications. So, no matter how many tables your project accesses, you will have 2 classes doing all the heavy lifting. That is 98% smaller than any comparable Spring Data based solution, which will have an entity, a repository and a REST service, at a minimum, FOR EACH TABLE.

    Getting Started

    1. Download the Tomato source.
    2. Unpack it into a new Java Project in whatever IDE you desire.
    3. Modify the EntityDataSource class so that it initializes a properly configured DataSource instance for the database you desire Tomato to connect to.
    4. Build the project into a jar file.
    5. Include the jar in the project of your choice.
    6. Check out the code in the EntityTest.java class. It includes examples of how to use all Tomato features.

    How easy is it, really?

    The following is the code needed to retrieve all records in any table...

    Entity e = Entity.createEntityOrView("Schema.tableName"); e.setFilter("1=1"); e.load();

    Want to load a hierarchy, like say, employees and their addresses? The following will load all employee records with nested addresses under each.

    1. Entity e = Entity.createEntityOrView("Schema.employees");
    2. e.setFilter("1=1");
    3. e.getChildList().add(new ChildData("schema.address", "address_id", false, false));
    4. e.load();

    If you are doing web development

    You can also download the companion project Basil. Basil provides dynamic forms and tables for any Tomato entity instance. Basil is written in Angular 1.5 and provides for simply injection of custom HTML where ever needed in a dynamic form or table.
