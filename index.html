    **Tomato is the ORM framework for the modern development world.** ============================================================= Tomato does more in less time and less code than Spring Data. ------- - When using Tomato... - you will not generate nor write entity classes. - You will not write repository classes. - You will not write service classes. - You will not write REST services. - You will not need to configure a persistence context. - You will not need to work with EntityManager instances. - You will not need to annotate your code - Using Tomato will reduce the size and complexity of your data accessing code by at least 98%. - An Entity and a REST service, both provided right out of the box, are all that you need to access any table in any schema, faster and with more flexibility than Spring Data. - Spring Data requires Controllers, Repositories, services and entities for every table or view you want to access. *Tomato is also completely thread safe without any modifications. * Tomato is a great solution for embedded systems - you won't find anything smaller and faster!

    Getting Started

    1. Download the Tomato source.
    2. Unpack it into a new Java Project in whatever IDE you desire.
    3. Modify the EntityDataSource class so that it initializes a properly configured DataSource instance for the database you desire Tomato to connect to.
    4. Build the project into a jar file.
    5. Include the jar in the project of your choice.
    6. Look below for sample code showing how to use Tomato, or check out the code in the EntityTest.java class for examples of all of Tomatos features.

    How easy is it, really?

    The following is the code needed to retrieve all records in any table... Entity e = Entity.createEntityOrView("Schema.tableName"); e.setFilter("1=1"); e.load(); Want to load and save a hierarchy? Like say, employees and their addresses? The following will load all employee records with nested addresses under each. Entity e = Entity.createEntityOrView("schema.employee"); e.setFilter("1=1"); ChildData cd = new ChildData("schema.address", "address_id", false, false); e.getChildList().add(cd); e.load();

    Are you building web forms for your data?

    If so, you might want to download Tomato's companion project, Basil. Basil provides dynamic forms and tables for any Tomato entity instance. Basil is written in Angular 1.5, and was designed for ease of customization. It supports injection of custom HTML anywhere needed inside a dynamically generated form or table. Using Basil, you can present an HTML table displaying all the rows in a given table, and which supports Adding, Editing and Deleting rows from that table, in 3 lines of HTML, inclusive - no custom code is needed on the server at all. See Basil at : https://rpbarbati.github.io/Basil
